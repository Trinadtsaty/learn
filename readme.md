Git команды

Commit (сохранение локально)
git add .              # добавляет все файлы
git commit -m "текст"  # создает коммит с сообщением

Push (отправка на GitHub)
git push               # отправляет изменения на сервер
git push -u origin main # первый пуш (устанавливает связь с веткой)

Pull (забрать с GitHub)
git pull               # скачать изменения с сервера

Остальные полезные команды
git status             # показывает текущее состояние (измененные файлы)
git log --oneline      # показывает историю коммитов (сжато)
git remote -v          # показывает связанный GitHub репозиторий
git branch             # показывает текущую ветку
git checkout -b name   # создает новую ветку и переключается на нее
git merge name         # сливает ветку name в текущую
git clone url          # скачать репозиторий с GitHub

Связь с GitHub
git remote add origin https://github.com/твой-логин/репозиторий.git



Git всегда работает в ТЕКУЩЕЙ папке, где открыт терминал.

То есть:

Ты открываешь терминал

Переходишь в папку проекта:

bash
cd G:\goProject\learn
И уже здесь выполняешь git status, git add, git commit

Как проверить, что ты в правильной папке:

bash
pwd            # показывает текущий путь (где ты)
ls             # показывает содержимое папки
git status     # показывает файлы именно этой папки
Если в git status ты видишь свои файлы/папки проекта — значит пушнется то, что нужно.
