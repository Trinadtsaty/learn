## 2.1. Переменные и константы

В этой теме мы заложим фундамент работы с данными в Go. Вы узнаете, как объявлять переменные и константы, какие типы данных существуют, как преобразовывать значения из одного типа в другой, а также познакомитесь с мощным инструментом `iota` для создания перечислений.

### 2.1.1. Переменные

Переменная — это именованная область памяти, в которой хранится значение определённого типа. В Go переменные всегда инициализированы (имеют значение), даже если вы явно не присвоили им значение — для этого существуют **нулевые значения**.

#### Объявление с помощью `var`

Самый явный способ объявить переменную — использовать ключевое слово `var`:
```go
var name type = value
```
Примеры:
```go
var age int = 30
var price float64 = 19.99
var active bool = true
var message string = "Hello"
```
Можно опустить инициализатор, тогда переменная получит **нулевое значение** для своего типа:

- Для чисел: `0`
- Для `bool`: `false`
- Для `string`: `""` (пустая строка)
- Для указателей, интерфейсов, срезов, каналов, карт: `nil`
```go
var count int       // 0
var isReady bool    // false
var text string     // ""
```
Можно объявить несколько переменных одного типа в одной строке:
```go
var x, y int = 10, 20
```
Или разные типы с использованием блока `var`:
```go
var (
    name   string = "Alice"
    age    int    = 25
    height float64
)
```
#### Короткое объявление `:=`

Внутри функций (не на уровне пакета) можно использовать краткую форму `:=`. Она одновременно объявляет переменную и присваивает ей значение, при этом тип выводится автоматически из выражения справа.
```go
func main() {
    name := "Bob"      // string
    age := 42          // int
    price := 99.5      // float64
    enabled := true    // bool
}
```
**Важные ограничения**:

- `:=` работает только внутри функций.
- Слева должна быть хотя бы одна новая переменная. Если переменная уже объявлена в той же области видимости, возникнет ошибка компиляции (но можно использовать `=` для присваивания существующей переменной).
- Используется для локальных переменных, когда тип очевиден.

#### Нулевые значения (zero values)

В Go нет неинициализированных переменных. Если вы объявили переменную без начального значения, ей автоматически присваивается нулевое значение для данного типа. Это важная особенность языка, которая предотвращает множество ошибок, связанных с неопределённым состоянием.
```go
var i int       // 0
var f float64   // 0.0
var b bool      // false
var s string    // ""
var p *int      // nil
```
#### Где хранятся переменные?

Локальные переменные функций обычно размещаются на **стеке** — области памяти, управляемой эффективно (выделение и освобождение происходит при входе/выходе из функции). Если переменная "убегает" из функции (например, её адрес возвращается или сохраняется в глобальной структуре), она может быть размещена в **куче** (heap). Это называется escape analysis. Пока нам достаточно знать, что локальные простые переменные, как правило, находятся на стеке, что быстро.
### 2.1.2. Подробнее о базовых типах данных

#### Целые числа `int`

`int` — знаковый целый тип, размер которого зависит от платформы: на 32-битных системах это 32 бита, на 64-битных — 64 бита. Если нужен точный размер, используйте `int32`, `int64`, а также беззнаковые `uint32`, `uint64`.
```go
var a int = 1000
var b int32 = 1000
var c int64 = 1000
```
#### Числа с плавающей точкой `float64`

`float64` — основной тип для чисел с плавающей точкой двойной точности (64 бита). `float32` используется реже, когда важна экономия памяти (например, в графике), но точность ниже.
```go
var pi float64 = 3.1415926535
```
#### Булев тип `bool`

Может принимать только значения `true` или `false`. В отличие от некоторых языков, в Go нельзя использовать числа в условных выражениях напрямую.

#### Строки `string`

Строки в Go — это неизменяемые последовательности байт, обычно представляющие текст в кодировке UTF-8. Неизменяемость означает, что после создания строку нельзя изменить (можно только создать новую на основе старой).
```go
var s string = "Привет, мир!"
```
### 2.1.3. Приведение типов

В Go нет автоматического приведения типов (как, например, в C или JavaScript). Даже если типы совместимы по смыслу, преобразование нужно выполнять явно с помощью синтаксиса `T(v)`, где `T` — целевой тип, а `v` — значение.
```go
var i int = 42
var f float64 = float64(i)   // int → float64
var u uint = uint(f)         // float64 → uint (дробная часть отбрасывается)

var original string = "hello"
var bytes []byte = []byte(original)  // string → []byte (копирует данные)
var back string = string(bytes)      // []byte → string (тоже копирует)
```
При преобразовании чисел с плавающей точкой в целые дробная часть отбрасывается (не округляется).
#### Разница между `int` и `uint` в Go
##### 1. Основное отличие: знаковость (signed vs unsigned)

###### `int` (signed integer — знаковое целое)
- Может хранить **как положительные, так и отрицательные** числа.
- Диапазон значений: от `-2^(n-1)` до `2^(n-1)-1`, где n — разрядность (32 или 64 бита).
- Пример: `-42`, `0`, `12345`
###### `uint` (unsigned integer — беззнаковое целое)
- Может хранить **только неотрицательные** числа (ноль и положительные).
- Диапазон значений: от `0` до `2^n - 1`.
- Пример: `0`, `42`, `999` (но не `-5`)
##### 2. Размер и платформозависимость

В Go оба типа являются **платформозависимыми**:
- На 32-битных системах: 32 бита (4 байта)
- На 64-битных системах: 64 бита (8 байт)
```go
package main

import (
    "fmt"
    "unsafe"
)

func main() {
    var i int
    var ui uint
    
    fmt.Printf("Размер int: %d байт\n", unsafe.Sizeof(i))   // 8 на 64-битной системе
    fmt.Printf("Размер uint: %d байт\n", unsafe.Sizeof(ui)) // тоже 8
}
```
Это означает, что размер `int` и `uint` может меняться в зависимости от архитектуры компьютера, где компилируется программа.
##### 3. Диапазоны значений (для 64-битной системы)
|Тип|Минимум|Максимум|
|---|---|---|
|int|-9 223 372 036 854 775 808|9 223 372 036 854 775 807|
|uint|0|18 446 744 073 709 551 615|
Как видите, `uint` может хранить **в два раза большие положительные числа**, чем `int`, за счёт отсутствия отрицательных значений.
##### 4. Когда что использовать?
###### Используйте `int` (знаковый), когда:
- **По умолчанию** — это наиболее естественный выбор для большинства задач.
- Значение может быть отрицательным (температура, разница, смещение).
- Вы работаете с циклами, индексами (хотя индексы неотрицательны, `int` удобнее из-за совместимости с функциями стандартной библиотеки).
- Вы не уверены, какой тип выбрать — `int` является идиоматичным выбором.
###### Используйте `uint` (беззнаковый), когда:
- Значение **никогда не может быть отрицательным** по логике предметной области (например, возраст, количество элементов, размер буфера).
- Вам нужен **максимальный положительный диапазон** (например, для битовых масок или криптографических операций).
- Вы работаете с двоичными форматами, сетевыми протоколами или системными вызовами, которые ожидают беззнаковые значения.
- Вы реализуете сложные битовые операции (сдвиги, маски), где беззнаковая арифметика ведёт себя предсказуемее.
##### 5. Важные предостережения при работе с `uint`
###### Опасность переполнения в меньшую сторону
Если вы вычитаете из меньшего беззнакового числа большее, произойдёт **переполнение (underflow)**, и результат станет очень большим положительным числом:
```go
var a uint = 5
var b uint = 10
result := a - b // Внимание! Не -5, а огромное число!
fmt.Println(result) // 18446744073709551611 (на 64-битной системе)
```
Компилятор **не предупредит** об этом, и программа может вести себя неожиданно.
###### Бесконечные циклы
```go
// НЕПРАВИЛЬНО - бесконечный цикл!
for i := uint(5); i >= 0; i-- {
    fmt.Println(i) // i никогда не станет меньше 0
}

// ПРАВИЛЬНО - используем int
for i := 5; i >= 0; i-- {
    fmt.Println(i)
}
```
###### Проблемы совместимости
Многие функции стандартной библиотеки Go работают с `int` (например, `len()` возвращает `int`). Если вы храните длину как `uint`, придётся постоянно преобразовывать:
```go
slice := []int{1, 2, 3}
var length uint = uint(len(slice)) // приходится приводить
```
##### 6. Фиксированные размеры
Если вам нужен точный размер независимо от платформы, используйте типы с явным размером:

|Знаковые|Беззнаковые|
|---|---|
|int8|uint8|
|int16|uint16|
|int32|uint32|
|int64|uint64|
##### 7. Практический пример
```go
package main

import "fmt"

func main() {
    // int - может быть отрицательным
    var temperature int = -5
    var balance int = -1000
    
    // uint - только неотрицательные
    var age uint = 30
    var count uint = 42
    
    fmt.Printf("Температура: %d\n", temperature)
    fmt.Printf("Баланс: %d\n", balance)
    fmt.Printf("Возраст: %d\n", age)
    fmt.Printf("Количество: %d\n", count)
    
    // Проблема с uint
    var x uint = 3
    var y uint = 5
    // result := x - y // РАСКОММЕНТИРУЙТЕ - получите огромное число!
    
    // Безопасный способ
    if x > y {
        result := x - y
        fmt.Println("Разность:", result)
    } else {
        fmt.Println("x меньше y, разность отрицательная, но uint не может её хранить")
    }
}
```
##### Пакета `unsafe`
**`unsafe`** — это специальный пакет в Go, который предоставляет возможности для низкоуровневой работы с памятью, **в обход системы типов Go**.
Название говорит само за себя: его использование **небезопасно** (unsafe), потому что нарушает гарантии типобезопасности, которые даёт Go. Компилятор не сможет защитить вас от ошибок, если вы используете этот пакет.
###### Почему он вообще существует?
Go — язык системного программирования. Иногда нужно:
- Взаимодействовать с C-кодом (cgo)
- Получать точный размер типов в байтах
- Работать с памятью на уровне указателей
- Оптимизировать критически важные участки кода
Для таких случаев и нужен `unsafe`.
### 2.1.4. Константы

Константы — это неизменяемые значения, которые вычисляются **на этапе компиляции**. Они могут быть только числами (целыми и с плавающей точкой), строками или булевыми значениями.

Объявляются с помощью `const`:
```go
const pi = 3.14159
const appName string = "MyApp"
```
Константы могут быть **типизированными** (как `appName` выше) и **нетипизированными** (как `pi`). Нетипизированные константы обладают гибкостью: они могут быть неявно преобразованы в любой совместимый тип при присваивании переменной.
```go
const x = 100          // нетипизированная
var y int = x          // ок, автоматически int
var z float64 = x      // ок, автоматически float64
```
#### iota — генератор последовательных значений

`iota` — это идентификатор, который используется внутри блоков `const` для создания последовательных целочисленных констант. Его значение начинается с 0 и увеличивается на 1 для каждой следующей строки в блоке.
```go
const (
    Sunday = iota  // 0
    Monday         // 1
    Tuesday        // 2
    Wednesday      // 3
    Thursday       // 4
    Friday         // 5
    Saturday       // 6
)
```
Можно использовать пропуски и выражения:
```go
const (
    _ = iota           // пропускаем 0
    KB = 1 << (10 * iota)  // 1 << 10 = 1024
    MB                     // 1 << 20 = 1048576
    GB                     // 1 << 30 = 1073741824
)
```
`iota` сбрасывается в 0 при входе в новый блок `const`.

### Практическое задание

**Задание:** Напишите программу, которая:

1. Объявляет переменные разных типов (`int`, `float64`, `bool`, `string`) разными способами (`var` без значения, `var` со значением, `:=`).
2. Выводит их нулевые значения (для тех, что не инициализированы) и присвоенные значения.
3. Выполняет приведение типов: преобразует `int` в `float64` и обратно, демонстрируя потерю дробной части.
4. Определяет константы с `iota` для дней недели (Пн, Вт, Ср, Чт, Пт, Сб, Вс) и выводит их числовые значения.
5. Запустите программу и покажите вывод.

**Решение (с подробными комментариями):**
```go
package main

import "fmt"

func main() {
    // 1. Объявление переменных разными способами

    // var без значения — получает нулевое значение
    var defaultInt int
    var defaultFloat float64
    var defaultBool bool
    var defaultString string

    // var со значением
    var explicitInt int = 42
    var explicitFloat float64 = 3.14
    var explicitBool bool = true
    var explicitString string = "Привет, Go!"

    // короткое объявление := (только внутри функций)
    shortInt := 100
    shortFloat := 2.718
    shortBool := false
    shortString := "короткая строка"

    // 2. Вывод нулевых и инициализированных значений
    fmt.Println("=== Нулевые значения (var без инициализации) ===")
    fmt.Printf("defaultInt: %v\n", defaultInt)       // 0
    fmt.Printf("defaultFloat: %v\n", defaultFloat)   // 0
    fmt.Printf("defaultBool: %v\n", defaultBool)     // false
    fmt.Printf("defaultString: %q\n", defaultString) // "" (пустая строка)

    fmt.Println("\n=== Явно инициализированные var ===")
    fmt.Printf("explicitInt: %v\n", explicitInt)
    fmt.Printf("explicitFloat: %v\n", explicitFloat)
    fmt.Printf("explicitBool: %v\n", explicitBool)
    fmt.Printf("explicitString: %q\n", explicitString)

    fmt.Println("\n=== Короткое объявление (:=) ===")
    fmt.Printf("shortInt: %v\n", shortInt)
    fmt.Printf("shortFloat: %v\n", shortFloat)
    fmt.Printf("shortBool: %v\n", shortBool)
    fmt.Printf("shortString: %q\n", shortString)

    // 3. Приведение типов
    fmt.Println("\n=== Приведение типов ===")
    var originalInt int = 123
    var asFloat float64 = float64(originalInt) // int -> float64
    var backToInt int = int(asFloat)           // float64 -> int (без потерь, т.к. целое)

    fmt.Printf("originalInt: %d, asFloat: %f, backToInt: %d\n", originalInt, asFloat, backToInt)

    // Демонстрация потери дробной части
    var pi float64 = 3.14159
    var intPi int = int(pi)
    fmt.Printf("pi = %f, int(pi) = %d (дробная часть отброшена)\n", pi, intPi)

    // 4. Константы с iota для дней недели
    const (
        Monday = iota + 1 // начинаем с 1, чтобы Пн = 1
        Tuesday
        Wednesday
        Thursday
        Friday
        Saturday
        Sunday
    )

    fmt.Println("\n=== Дни недели (iota) ===")
    fmt.Printf("Пн: %d, Вт: %d, Ср: %d, Чт: %d, Пт: %d, Сб: %d, Вс: %d\n",
        Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday)

    // Можно также вывести названия, но в этом задании только числа
}
```
**Ожидаемый вывод (с небольшими вариациями):**
```text

defaultInt: 0
defaultFloat: 0
defaultBool: false
defaultString: ""

=== Явно инициализированные var ===
explicitInt: 42
explicitFloat: 3.14
explicitBool: true
explicitString: "Привет, Go!"

=== Короткое объявление (:=) ===
shortInt: 100
shortFloat: 2.718
shortBool: false
shortString: "короткая строка"

=== Приведение типов ===
originalInt: 123, asFloat: 123.000000, backToInt: 123
pi = 3.141590, int(pi) = 3 (дробная часть отброшена)

=== Дни недели (iota) ===
Пн: 1, Вт: 2, Ср: 3, Чт: 4, Пт: 5, Сб: 6, Вс: 7
```
**Пояснения к решению:**

- Мы использовали `fmt.Printf` с спецификаторами `%v` (значение по умолчанию) и `%q` (для строк с кавычками), чтобы наглядно показать значения.
- Для демонстрации нулевых значений объявили переменные через `var` без инициализации.
- В разделе приведения показали, что преобразование `float64` в `int` отбрасывает дробную часть, а не округляет.
- Для дней недели применили `iota + 1`, чтобы начать отсчёт с 1 (обычно программисты предпочитают понедельник = 1, но можно и с 0, если нужно).
- Все константы нетипизированные, поэтому они могут быть использованы как числа.

## 2.3. Функции
Функции — это основные строительные блоки любой программы на Go. Они позволяют группировать код, переиспользовать его и строить абстракции. В этом разделе мы разберём все аспекты работы с функциями: от простого объявления до продвинутых концепций вроде замыканий и отложенных вызовов.
### 2.3.1. Базовое объявление функций

#### Синтаксис

В Go функция объявляется с помощью ключевого слова `func`, за которым следует имя функции, список параметров в круглых скобках, тип возвращаемого значения (если есть) и тело функции в фигурных скобках.
```go
func имя(параметры) возвращаемый_тип {
    // тело функции
    return значение
}
```
#### Простой пример
```go
func add(a int, b int) int {
    return a + b
}
```
Если несколько параметров имеют один тип, можно указать его один раз в конце:
```go
func add(a, b int) int {  // a и b оба int
    return a + b
}
```
#### Что происходит при вызове функции?

При вызове функции создаётся **фрейм стека** (stack frame) — область памяти в стеке, где хранятся:
- аргументы функции (копируются значения)
- локальные переменные
- адрес возврата (куда перейти после завершения функции)
Когда функция завершается, фрейм стека уничтожается, и управление возвращается вызывающему коду. Это основа работы с памятью в Go: локальные переменные живут только во время выполнения функции, если только компилятор не решит разместить их в куче (escape analysis).
### 2.3.2. Множественные возвращаемые значения

Одна из характерных особенностей Go — возможность возвращать из функции несколько значений. Это особенно удобно для возврата результата и ошибки одновременно.
```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("деление на ноль")
    }
    return a / b, nil
}
```
Использование:
```go
result, err := divide(10, 2)
if err != nil {
    fmt.Println("Ошибка:", err)
} else {
    fmt.Println("Результат:", result)
}
```
Этот паттерн стал идиоматическим в Go: функция возвращает полезное значение и ошибку. Ошибка `nil` означает успех.
### 2.3.3. Именованные результаты

В Go можно дать имена возвращаемым значениям. Тогда внутри функции они ведут себя как обычные переменные, а в конце достаточно написать `return` без аргументов (так называемый "голый return").
```go
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return // автоматически вернёт x и y
}
```
**Важно:** "голый return" может ухудшить читаемость в длинных функциях. В коротких функциях это приемлемо. Старайтесь использовать именованные результаты, когда они действительно упрощают код (например, при работе с defer, когда нужно изменить возвращаемое значение).
### 2.3.4. Функции как значения первого класса

В Go функции являются **значениями первого класса** (first-class citizens). Это означает, что их можно:
- присваивать переменным
- передавать в другие функции как аргументы
- возвращать из функций

Тип функции определяется её сигнатурой: `func(параметры) возвращаемые_типы`.
```go
package main

import "fmt"

// Определяем функцию add
func add(a int, b int) int {
    return a + b
}

func main() {
    // Объявляем переменную типа "функция, принимающая два int и возвращающая int"
    var operation func(int, int) int
    
    // Присваиваем ей функцию add
    operation = add
    
    // Теперь можем вызывать функцию через переменную
    result := operation(5, 3)
    
    fmt.Println(result) // Вывод: 8
}
```
#### Передача функций как параметров

Это основа для создания гибких API и реализации паттернов вроде стратегии:
```go
func applyOperation(a, b int, op func(int, int) int) int {
    return op(a, b)
}

func add(a, b int) int { return a + b }
func mul(a, b int) int { return a * b }

fmt.Println(applyOperation(5, 3, add)) // 8
fmt.Println(applyOperation(5, 3, mul)) // 15
```
### 2.3.5. Анонимные функции и замыкания

#### Анонимные функции

Функции без имени называются анонимными. Их можно объявлять прямо в месте использования.
```go
// Присваивание переменной
increment := func(x int) int {
    return x + 1
}
fmt.Println(increment(5)) // 6

// Немедленный вызов
max := func(a, b int) int {
    if a > b { return a }
    return b
}(10, 20) // сразу вызываем с аргументами
fmt.Println(max) // 20
```
#### Замыкания (closures)

**Замыкание** — это анонимная функция, которая "захватывает" переменные из внешней области видимости (из окружающей функции). Замыкание "помнит" состояние этих переменных даже после того, как внешняя функция завершилась.
```go
func makeCounter() func() int {
    count := 0
    return func() int {
        count++ // захватывает count из внешней функции
        return count
    }
}

counter1 := makeCounter()
fmt.Println(counter1()) // 1
fmt.Println(counter1()) // 2
fmt.Println(counter1()) // 3

counter2 := makeCounter() // новый независимый счётчик
fmt.Println(counter2())   // 1
```
**Как это работает под капотом:**  
Переменная `count` обычно должна была бы жить только во время вызова `makeCounter`, но компилятор обнаруживает, что на неё ссылается возвращаемая функция. В результате `count` "убегает" в кучу (escape analysis) и продолжает существовать, пока существует замыкание. Каждый вызов `makeCounter` создаёт новую переменную `count` в куче, независимую от других.

Замыкания полезны для:
- создания фабрик функций
- инкапсуляции состояния (счётчики, кеши)
- реализации колбэков с контекстом

### 2.3.6. Отложенный вызов defer

`defer` откладывает выполнение функции до выхода из окружающей функции. Это мощный механизм для гарантированной очистки ресурсов.
```go
func readFile(filename string) error {
    f, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer f.Close() // гарантированно закроет файл при выходе из функции
    
    // работа с файлом...
    return nil
}
```
#### Порядок выполнения

Если в функции несколько `defer`, они выполняются в порядке **LIFO** (Last In, First Out) — последний отложенный вызов выполнится первым.
```go
func main() {
    defer fmt.Println("первый defer")
    defer fmt.Println("второй defer")
    defer fmt.Println("третий defer")
    fmt.Println("обычный код")
}
// Вывод:
// обычный код
// третий defer
// второй defer
// первый defer
```
#### Важный нюанс: аргументы defer вычисляются сразу

Аргументы отложенной функции вычисляются в момент объявления `defer`, а не в момент выполнения.
```go
func main() {
    x := 10
    defer fmt.Println("x в defer:", x) // вычислится сейчас, x = 10
    x = 20
    fmt.Println("x в main:", x)        // x = 20
}
// Вывод:
// x в main: 20
// x в defer: 10
```
Если нужно передать актуальное значение, используйте замыкание:
```go
x := 10
defer func() {
    fmt.Println("x в defer через замыкание:", x) // захватывает x
}()
x = 20
// Вывод: x в defer через замыкание: 20
```
#### Как это работает:
1. **Анонимная функция** `func() { ... }` создается
2. Она **замыкает** переменную `x` из внешней области видимости
3. Функция запоминает **ссылку** на переменную `x`, а не её значение на момент создания
4. Когда функция выполняется (после `x = 20`), она использует **текущее** значение `x`
#### Типичные применения defer
- Закрытие файлов, сетевых соединений
- Освобождение мьютексов (`defer mu.Unlock()`)
- Замер времени выполнения функции
- Восстановление после паники (recover)

#### Defer и recover (кратко)

`defer` может использоваться с `recover` для обработки паники:
```go
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Восстановились после паники:", r)
        }
    }()
    panic("что-то пошло не так") // программа не упадёт
}
```


### Практическое задание

**Задание:** Напишите программу, которая:
1. Создаёт функцию `divide(a, b float64) (float64, error)`, которая возвращает ошибку при делении на ноль.
2. Создаёт функцию `makeMultiplier(multiplier float64) func(float64) float64`, которая возвращает замыкание, умножающее аргумент на multiplier.
3. В `main()`:
    - Вызовите `divide` с разными аргументами (включая деление на ноль) и обработайте ошибку.
    - Создайте умножитель на 2 с помощью `makeMultiplier` и примените его к нескольким числам.
    - Используйте `defer` для вывода сообщения "Завершение программы" в самом начале `main`.
    - Покажите работу `defer`: объявите несколько `defer` в `main` и убедитесь, что они выполняются в обратном порядке.
    - Используйте анонимную функцию с немедленным вызовом для инициализации значения (например, максимального числа из двух).
**Решение (с подробными комментариями):**
```go
package main

import (
    "fmt"
    "errors"
)

// 1. Функция с множественным возвратом: значение и ошибка
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("деление на ноль запрещено")
    }
    return a / b, nil
}

// 2. Функция, возвращающая замыкание (фабрика умножителей)
func makeMultiplier(multiplier float64) func(float64) float64 {
    // Возвращаем анонимную функцию, которая "захватила" multiplier
    return func(x float64) float64 {
        return x * multiplier
    }
}

func main() {
    // Демонстрация defer: порядок LIFO
    defer fmt.Println("1. Первый defer (но выполнится последним из defer)")
    defer fmt.Println("2. Второй defer (выполнится предпоследним)")
    defer fmt.Println("3. Третий defer (выполнится первым среди defer)")
    
    // Самый первый defer в начале main — для сообщения о завершении
    defer fmt.Println("Завершение программы (этот defer объявлен первым, но выполнится последним)")

    fmt.Println("=== Демонстрация divide ===")
    
    // Вызов divide с корректными аргументами
    result1, err := divide(10, 2)
    if err != nil {
        fmt.Printf("Ошибка: %v\n", err)
    } else {
        fmt.Printf("10 / 2 = %.2f\n", result1)
    }
    
    // Вызов divide с делением на ноль
    result2, err := divide(5, 0)
    if err != nil {
        fmt.Printf("Ошибка при делении 5/0: %v\n", err)
    } else {
        fmt.Printf("5 / 0 = %.2f\n", result2) // этот код не выполнится
    }
    
    fmt.Println("\n=== Демонстрация makeMultiplier ===")
    
    // Создаём умножитель на 2
    double := makeMultiplier(2)
    
    // Создаём умножитель на 10
    timesTen := makeMultiplier(10)
    
    // Применяем замыкания
    fmt.Printf("double(3) = %.2f\n", double(3))
    fmt.Printf("double(7.5) = %.2f\n", double(7.5))
    fmt.Printf("timesTen(3) = %.2f\n", timesTen(3))
    fmt.Printf("timesTen(1.5) = %.2f\n", timesTen(1.5))
    
    // Проверяем, что замыкания независимы
    fmt.Printf("Ещё раз double(4) = %.2f\n", double(4))
    
    fmt.Println("\n=== Демонстрация анонимной функции с немедленным вызовом ===")
    
    // Анонимная функция для вычисления максимума
    max := func(a, b int) int {
        if a > b {
            return a
        }
        return b
    }(45, 78) // немедленный вызов
    
    fmt.Printf("Максимальное из 45 и 78 = %d\n", max)
    
    // Ещё один пример: инициализация сложного значения
    greeting := func(name string) string {
        return "Привет, " + name + "!"
    }("Алиса")
    
    fmt.Println(greeting)
    
    // Дополнительная демонстрация: defer с вычислением аргументов
    fmt.Println("\n=== Демонстрация вычисления аргументов defer ===")
    x := 10
    defer fmt.Println("Значение x в defer (вычислено сейчас):", x) // сохранит 10
    x = 20
    fmt.Println("Значение x в main:", x)
    
    // Чтобы получить актуальное значение, используем замыкание
    y := 10
    defer func() {
        fmt.Println("Значение y в defer через замыкание:", y) // возьмёт текущее y
    }()
    y = 20
    fmt.Println("Значение y в main:", y)
}
```
**Ожидаемый вывод (порядок может немного отличаться, но суть та же):**
```text
=== Демонстрация divide ===
10 / 2 = 5.00
Ошибка при делении 5/0: деление на ноль запрещено

=== Демонстрация makeMultiplier ===
double(3) = 6.00
double(7.5) = 15.00
timesTen(3) = 30.00
timesTen(1.5) = 15.00
Ещё раз double(4) = 8.00

=== Демонстрация анонимной функции с немедленным вызовом ===
Максимальное из 45 и 78 = 78
Привет, Алиса!

=== Демонстрация вычисления аргументов defer ===
Значение x в main: 20
Значение y в main: 20
Значение x в defer (вычислено сейчас): 10
Значение y в defer через замыкание: 20
3. Третий defer (выполнится первым среди defer)
4. Второй defer (выполнится предпоследним)
5. Первый defer (но выполнится последним из defer)
Завершение программы (этот defer объявлен первым, но выполнится последним)
```
**Пояснения к ключевым моментам:**

- **`divide` возвращает ошибку** — это идиоматичный способ обработки ошибок в Go.
- **`makeMultiplier` возвращает замыкание** — каждая переменная `double` и `timesTen` хранит свой собственный `multiplier` (2 и 10 соответственно). Это достигается благодаря захвату переменной из внешней функции.
- **Порядок defer** — мы видим, что `defer` выполняются в обратном порядке относительно их объявления. Самый последний в коде (но объявленный первым) `defer fmt.Println("Завершение программы")` выполняется в самом конце.
- **Вычисление аргументов defer** — пример с `x` показывает, что аргументы фиксируются в момент объявления `defer`. Пример с `y` через замыкание показывает, как получить актуальное значение.
- **Анонимная функция с немедленным вызовом** — удобна для инициализации значений, требующих логики, без загрязнения области видимости лишними именами.

Это задание охватывает все ключевые концепции работы с функциями в Go. В следующей теме мы перейдём к пакетам и импортам, чтобы научиться организовывать код в больших проектах.

## 2.4. Пакеты и импорты
Пакеты — это способ организации и структурирования кода в Go. Они обеспечивают модульность, инкапсуляцию и переиспользование. Понимание пакетов критически важно для создания любых нетривиальных программ.

### 2.4.1. Структура пакета

**Пакет** — это набор файлов с расширением `.go`, находящихся в одной директории. Все файлы в этой директории должны начинаться с одного и того же объявления `package`. Исключение составляют тестовые файлы с суффиксом `_test`, которые могут принадлежать отдельному пакету для тестирования.
#### Соглашение об именовании
Хотя имя пакета может быть любым (кроме "main"), существует соглашение: **имя пакета совпадает с именем директории**, в которой он находится. Это упрощает навигацию и импорт.
#### Пример структуры проекта
```text
myproject/
├── go.mod
├── main.go
└── math/
    ├── math.go
    └── utils.go
```
- `main.go` принадлежит пакету `main` (особый пакет для исполняемых программ)
- `math/math.go` и `math/utils.go` принадлежат пакету `math` (оба файла начинаются с `package math`)
### 2.4.2. Экспортируемые и неэкспортируемые идентификаторы

В Go видимость за пределами пакета определяется **регистром первой буквы** идентификатора:
- **С заглавной буквы** — экспортируемый (публичный) идентификатор. Доступен из других пакетов.
- **Со строчной буквы** — неэкспортируемый (приватный) идентификатор. Доступен только внутри своего пакета.
Это относится ко всем идентификаторам: переменным, константам, функциям, типам, полям структур.
#### Пример
Создадим пакет `math` в файле `math/math.go`:
```go
package math

import "fmt"

// Add — экспортируемая функция (доступна извне)
func Add(a, b int) int {
    logOperation("Add") // вызываем внутреннюю функцию
    return a + b
}

// sub — неэкспортируемая функция (только внутри пакета)
func sub(a, b int) int {
    logOperation("sub")
    return a - b
}

// logOperation — неэкспортируемая вспомогательная функция
func logOperation(op string) {
    fmt.Printf("Выполняется операция: %s\n", op)
}
```
В `main.go`:
```go
package main

import (
    "fmt"
    "myproject/math" // путь относительно корня модуля
)

func main() {
    sum := math.Add(5, 3)
    fmt.Println("5 + 3 =", sum)

    // sub не видна — ошибка компиляции:
    // diff := math.sub(5, 3) // cannot refer to unexported name math.sub
}
```
### 2.4.3. Импорт пакетов

#### Стандартный импорт
```go
import "fmt"
import "math/rand"
```
Удобнее группировать:
```go
import (
    "fmt"
    "math/rand"
)
```
#### Путь импорта
- Для стандартных пакетов: просто имя (`"fmt"`, `"net/http"`)
- Для внешних пакетов: полный URL (`"github.com/gorilla/mux"`)
- Для внутренних пакетов: от корня модуля (`"myproject/math"`)

#### Импорт с псевдонимом
Когда нужно избежать конфликта имён или сократить часто используемое имя:
```go
import f "fmt"

func main() {
    f.Println("Hello with alias")
}
```
#### Точечный импорт (не рекомендуется)
```go
import . "fmt"

func main() {
    Println("Hello without prefix") // Println доступен без префикса
}
```
**Почему не рекомендуется в production:**
- Загрязняет пространство имён (непонятно, откуда взялась функция)
- Ухудшает читаемость и поддержку кода
- Может привести к конфликтам имён
#### Пустой импорт
```go
import _ "image/png"
```
Пустой импорт используется, когда нужно выполнить **побочный эффект** импорта — обычно вызов функции `init()` в импортируемом пакете, но сам пакет напрямую не используется.

Типичные примеры:
- Регистрация драйверов баз данных: `import _ "github.com/go-sql-driver/mysql"`
- Регистрация форматов изображений: `import _ "image/png"`
- Регистрация парсеров конфигураций

### 2.4.4. Функция init()
`init()` — это специальная функция без параметров и возвращаемого значения. Она не может быть вызвана явно, а выполняется автоматически при импорте пакета.
#### Порядок инициализации пакета
Когда пакет импортируется, Go выполняет следующие шаги в строгом порядке:
1. **Инициализация импортированных пакетов** (рекурсивно)
2. **Вычисление констант** на уровне пакета
3. **Инициализация переменных** на уровне пакета
4. **Вызов функций `init()`** в порядке их появления в файлах (сначала в одном файле, потом в других по алфавиту)
#### Пример с несколькими init()
```go
package math

import "fmt"

var Pi = 3.14159

func init() {
    fmt.Println("Первый init в math")
}

func init() {
    fmt.Println("Второй init в math")
}
```
При импорте этого пакета сначала инициализируется `Pi`, затем выполнятся оба `init()` по порядку.

#### Предостережение

`init()` усложняет тестирование и делает поведение программы менее явным. Используйте их только для действительно необходимой инициализации (например, регистрация драйверов, проверка конфигурации), а не для бизнес-логики.
### 2.4.5. Циклические импорты
**Циклический импорт** возникает, когда пакет A импортирует пакет B, а пакет B (напрямую или через цепочку) импортирует пакет A.
```go
// package a
import "myproject/b" // ОШИБКА: циклический импорт, если b импортирует a

// package b
import "myproject/a"
```
Компилятор Go запрещает циклические импорты. Это заставляет проектировать архитектуру с чёткими направлениями зависимостей.
**Как исправить:**
- Вынести общий код в третий пакет
- Использовать интерфейсы для инверсии зависимостей
- Пересмотреть архитектуру, возможно, объединить пакеты

### 2.4.6. Как работает компиляция пакетов "под капотом"

Go компилирует пакеты **независимо** и **параллельно**, что обеспечивает высокую скорость сборки. Каждый пакет компилируется в объектный файл (`.a`), которые затем линкуются в исполняемый файл.

**Кэширование:** Результаты компиляции пакетов кэшируются (в `$GOPATH/pkg/mod` или `$GOCACHE`). При повторной сборке перекомпилируются только изменившиеся пакеты и их зависимые.

**Почему нельзя импортировать неиспользуемые пакеты?**

- Философия Go: чистота кода и явность. Неиспользуемый импорт — это "мёртвый груз", который может скрывать проблемы.
- Технически: компилятор проверяет все импорты, и если пакет импортирован, но не используется, это считается ошибкой. Это ускоряет компиляцию и предотвращает случайные зависимости.

## 2.5. Обработка ошибок (начало)

В этом разделе мы познакомимся с фундаментальным подходом Go к обработке ошибок. В отличие от многих языков, где используются исключения (exceptions), в Go ошибки — это обычные значения, которые функция может возвращать и которые программист обрабатывает явно. Такой подход делает поток выполнения программы более предсказуемым и прозрачным.

### 2.5.1. Концепция "ошибки как значения"

**В Go нет исключений.** Вместо этого функции, которые могут завершиться неудачей, возвращают дополнительное значение — ошибку. Обычно это последнее возвращаемое значение, и его тип — `error`. Например, функция деления может вернуть результат и ошибку:
```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("деление на ноль")
    }
    return a / b, nil
}
```
**Философия:** программист **явно** обрабатывает ошибки сразу после вызова функции. Нет скрытых механизмов, которые прерывают нормальный ход программы (как исключения). Это делает код более читаемым и надёжным, так как все места, где что-то может пойти не так, видны и требуют внимания.
### 2.5.2. Тип error
`error` — это встроенный интерфейс. Его определение выглядит так:
```go
type error interface {
    Error() string
}
```
Любой тип, который реализует метод `Error() string`, автоматически удовлетворяет интерфейсу `error`. То есть вы можете создать свой тип ошибки с дополнительной информацией, и его можно будет использовать везде, где ожидается `error`.

**Нулевое значение интерфейса** — это `nil`. Интерфейс в Go представляет собой пару: **(тип, значение)**. Он равен `nil` только тогда, когда и тип, и значение равны `nil`. Если интерфейсу присвоено значение конкретного типа, даже если это значение `nil` (например, указатель `nil`), интерфейс **не будет равен `nil`**, потому что тип теперь известен.

Пример, иллюстрирующий структуру интерфейса (упрощённо):
- Пустой интерфейс `nil`: (nil, nil)
- Интерфейс, хранящий указатель на `os.PathError`, равный nil: (*os.PathError, nil) — не равен nil.
Это важный нюанс, к которому мы вернёмся.
#### Создание своего типа ошибки (для иллюстрации)
```go
type MyError string

func (e MyError) Error() string {
    return string(e)
}

func doSomething() error {
    return MyError("что-то не так")
}
```
### 2.5.3. Создание ошибок
#### Пакет `errors`
Функция `errors.New` создаёт новую ошибку с заданным текстом:
```go
import "errors"

err := errors.New("недостаточно памяти")
```
Внутри `errors.New` возвращает указатель на неэкспортируемый тип `errorString`, который хранит строку и реализует метод `Error()`.
#### Пакет `fmt` — `fmt.Errorf`
Функция `fmt.Errorf` форматирует строку и возвращает `error`. Это удобно, когда нужно включить в сообщение динамические данные:
```go
import "fmt"

filename := "config.txt"
err := fmt.Errorf("ошибка открытия файла %s: %v", filename, os.ErrNotExist)
```
Обычно `fmt.Errorf` также возвращает `errorString`, но может возвращать и другие типы, если используются специальные форматирующие глаголы (например, `%w` для обёртывания ошибок, но это тема продвинутого уровня).
### 2.5.4. Простейшая проверка `if err != nil`

После вызова функции, которая возвращает ошибку, принято сразу проверять её:
```go
result, err := divide(10, 0)
if err != nil {
    // Обрабатываем ошибку
    fmt.Println("Ошибка:", err)
    return // или продолжаем, но с учётом ошибки
}
fmt.Println("Результат:", result)
```
**Почему именно `err != nil`?**  
При успешном выполнении функция возвращает `nil` в качестве ошибки. `nil` — это нулевое значение интерфейса. Проверка `err != nil` истинна, если интерфейс не пустой (т.е. хранит какой-то тип и значение).

**Важно:** никогда не игнорируйте ошибки без необходимости. Если вы уверены, что ошибки быть не может, можно явно присвоить её `_`:
```go
result, _ := divide(10, 2) // игнорируем ошибку (не рекомендуется)
```
Но лучше хотя бы залогировать её или обработать.
### 2.5.5. Особенности и подводные камни

#### Ошибка может быть не nil, даже если значение nil, но тип не nil
Рассмотрим пример с пользовательским типом ошибки, который является указателем:

```go
type MyError struct {
    Msg string
}

func (e *MyError) Error() string {
    return e.Msg
}

func returnsError() error {
    var err *MyError = nil
    return err
}

func main() {
    err := returnsError()
    if err != nil {
        fmt.Println("Ошибка не nil!") // Этот код выполнится!
    }
}
```
Почему? Переменная `err` в `returnsError` имеет конкретный тип `*MyError` и значение `nil`. Когда мы возвращаем её как `error`, интерфейс получает тип `*MyError` и значение `nil`. Интерфейс не равен `nil`, потому что его тип не `nil`. Это классическая ловушка в Go.

**Рекомендация:** всегда возвращать конкретный `nil` при успехе, а не `nil` конкретного типа. То есть:
```go
func returnsError() error {
    if someCondition {
        return &MyError{Msg: "bad"}
    }
    return nil // именно nil, а не переменная типа *MyError со значением nil
}
```
Если нужно вернуть переменную конкретного типа, которая может быть `nil`, убедитесь, что вы возвращаете её, только если она не `nil`, иначе возвращайте явный `nil`.
### 2.5.6. Множественные ошибки (кратко)

Иногда функция может вернуть несколько ошибок, но это редко. Обычно функция возвращает либо полезный результат и `nil`, либо нулевое значение результата и ошибку. Если нужно передать несколько ошибок, можно вернуть слайс ошибок или использовать специальные пакеты (например, `multierror`), но это выходит за рамки начального уровня.

### Практическое задание

**Задание:** Напишите программу, которая демонстрирует базовую обработку ошибок в Go.

1. Создайте функцию `safeDivide(a, b float64) (float64, error)`, которая:

3. В `main()`:

    - Попросите пользователя ввести имя файла (или задайте тестовое имя, например "test.txt"), прочитайте его через `readFileContent`, выведите содержимое или ошибку.
    - Создайте свою простую ошибку с помощью `fmt.Errorf`, например, при проверке, что число отрицательное (можно добавить в `main`).

**Решение (с подробными комментариями):**
```go
package main

import (
    "errors"
    "fmt"
    "os"
)

// 1. Функция safeDivide возвращает результат и ошибку
func safeDivide(a, b float64) (float64, error) {
    if b == 0 {
        // Создаём ошибку с помощью errors.New
        return 0, errors.New("деление на ноль")
    }
    return a / b, nil // nil означает "ошибки нет"
}

// 2. Функция readFileContent читает файл и возвращает его содержимое как строку
func readFileContent(filename string) (string, error) {
    // os.ReadFile возвращает []byte и error
    data, err := os.ReadFile(filename)
    if err != nil {
        // Возвращаем пустую строку и полученную ошибку (например, файл не найден)
        return "", err
    }
    // Преобразуем []byte в string
    return string(data), nil
}

func main() {

    fmt.Println("=== safeDivide ===")
    a, b := 10.0, 0.0
    result, err := safeDivide(a, b)
    if err != nil {
        fmt.Printf("Ошибка при делении %.2f на %.2f: %v\n", a, b, err)
    } else {
        fmt.Printf("%.2f / %.2f = %.2f\n", a, b, result)
    }

    a, b = 15.0, 3.0
    result, err = safeDivide(a, b)
    if err != nil {
        fmt.Printf("Ошибка: %v\n", err)
    } else {
        fmt.Printf("%.2f / %.2f = %.2f\n", a, b, result)
    }

    // ========== Демонстрация readFileContent ==========
    fmt.Println("\n=== readFileContent ===")
    filename := "test.txt" // можно заменить на ввод пользователя, но для примера зададим
    // Создадим тестовый файл, если его нет
    _ = os.WriteFile(filename, []byte("Hello, file!\n"), 0644)

    content, err := readFileContent(filename)
    if err != nil {
        fmt.Printf("Ошибка чтения файла %s: %v\n", filename, err)
    } else {
        fmt.Printf("Содержимое файла %s:\n%s", filename, content)
    }

    // Попробуем прочитать несуществующий файл
    content, err = readFileContent("missing.txt")
    if err != nil {
        fmt.Printf("Ошибка чтения missing.txt: %v\n", err)
    }

    // ========== Создание ошибки через fmt.Errorf ==========
    fmt.Println("\n=== fmt.Errorf ===")
    age := -5
    if age < 0 {
        err := fmt.Errorf("возраст не может быть отрицательным: %d", age)
        fmt.Println(err)
    }

    // ========== Демонстрация ловушки с nil конкретного типа ==========
    fmt.Println("\n=== Ловушка: nil конкретного типа, присвоенный интерфейсу ===")
    var pathErr *os.PathError = nil // указатель типа *os.PathError, значение nil
    var errInterface error = pathErr // присваиваем интерфейсу error

    if errInterface != nil {
        // Этот блок выполнится, хотя значение pathErr было nil!
        fmt.Println("errInterface НЕ равен nil! (потому что тип *os.PathError присутствует)")
    } else {
        fmt.Println("errInterface равен nil")
    }

    // Чтобы избежать этого, всегда возвращайте явный nil для интерфейса:
    if errInterface == nil {
        fmt.Println("Это не выполнится, но если бы мы вернули nil напрямую, было бы ок.")
    }

    // Правильный способ: возвращать nil конкретного типа только если он действительно не nil,
    // иначе возвращать просто nil.
    // Пример правильной функции:
    func returnsNil() error {
        // some condition
        return nil // возвращаем конкретный nil интерфейса
    }

    err = returnsNil()
    if err == nil {
        fmt.Println("returnsNil вернула nil, как и ожидалось.")
    }
}
```
**Ожидаемый вывод:**
```text
=== safeDivide ===
Ошибка при делении 10.00 на 0.00: деление на ноль
15.00 / 3.00 = 5.00

=== readFileContent ===
Содержимое файла test.txt:
Hello, file!
Ошибка чтения missing.txt: open missing.txt: no such file or directory

=== fmt.Errorf ===
возраст не может быть отрицательным: -5

=== Ловушка: nil конкретного типа, присвоенный интерфейсу ===
errInterface НЕ равен nil! (потому что тип *os.PathError присутствует)
returnsNil вернула nil, как и ожидалось.
```
**Пояснения к ключевым моментам:**
- **`errors.New`** создаёт простую ошибку с фиксированным текстом.
- **`fmt.Errorf`** позволяет форматировать сообщение ошибки.
- **`os.ReadFile`** демонстрирует типичную функцию, возвращающую ошибку (если файл не существует).
- **Ловушка с nil конкретного типа:** показано, что интерфейс, содержащий указатель `nil` конкретного типа, не равен `nil`. Это важный момент для избежания труднонаходимых багов.
- **Правильный возврат ошибок:** функция `returnsNil` демонстрирует, что нужно возвращать именно `nil`, а не переменную конкретного типа со значением `nil`, чтобы интерфейс был действительно `nil`.

Теперь вы знаете основы обработки ошибок в Go. В следующей теме мы перейдём к составным типам данных: массивам и слайсам.
### 2.5.7. Обёртывание ошибок (error wrapping)

Начиная с Go 1.13, в язык добавлена возможность **оборачивать** ошибки, добавляя контекст, но сохраняя доступ к исходной ошибке. Это делается с помощью `fmt.Errorf` с глаголом `%w`.

**Пример:**
```go
func readConfig(filename string) error {
    data, err := os.ReadFile(filename)
    if err != nil {
        return fmt.Errorf("read config %s: %w", filename, err) // оборачиваем
    }
    // ...
    return nil
}
```
Теперь вызвавшая функция может проверить исходную ошибку с помощью `errors.Is` или `errors.As`.
#### Для чего это нужно?

- Добавлять контекст к ошибке (например, "не удалось открыть файл конфигурации").
- Не терять возможность определить конкретный тип ошибки (например, `os.ErrNotExist`).
#### Проверка обёрнутых ошибок

Пакет `errors` предоставляет две основные функции:

- **`errors.Is(err, target error) bool`** — проверяет, что ошибка `err` (или любая ошибка в цепочке) равна `target`. Учитывает обёртки.
- **`errors.As(err error, target any) bool`** — находит в цепочке первую ошибку, тип которой соответствует типу `target`, и присваивает её `target`. Позволяет получить доступ к конкретному типу ошибки с дополнительными полями.

**Пример:**
```go
err := readConfig("missing.json")

if errors.Is(err, os.ErrNotExist) {
    fmt.Println("Файл не существует")
}

var pathError *os.PathError
if errors.As(err, &pathError) {
    fmt.Println("Путь:", pathError.Path)
}
```
Без обёртывания `fmt.Errorf` с `%w` исходная ошибка была бы потеряна.
### 2.5.8. Сентинельные ошибки (sentinel errors)

Это **предопределённые ошибки**, которые можно сравнивать напрямую (обычно они объявлены как переменные). Например:

- `io.EOF` — конец файла.
- `os.ErrNotExist` — файл не существует.
- `sql.ErrNoRows` — нет строк в результате запроса.

Их часто используют для сигнализации о конкретных ожидаемых ситуациях. Сравнивать их можно с помощью `==`, но лучше использовать `errors.Is`, так как при обёртывании прямое сравнение не сработает.

**Пример:**
```go
err := someFunction()
if err == io.EOF {
    // конец файла
}
```
Но более идиоматично:
```go
if errors.Is(err, io.EOF) {
    // работает даже если err обёрнута
}
```
### 2.5.9. Кастомные типы ошибок

Вы можете создавать собственные типы, реализующие интерфейс `error`. Это полезно, когда нужно передать дополнительную информацию, например код ошибки, временную метку или структурированные данные.

**Пример:**
```go
type ValidationError struct {
    Field string
    Value interface{}
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation failed on field %s: %s", e.Field, e.Message)
}

func validateAge(age int) error {
    if age < 0 {
        return &ValidationError{Field: "age", Value: age, Message: "negative age"}
    }
    return nil
}
```
Затем в вызывающем коде можно использовать `errors.As`, чтобы получить доступ к полям:
```go
err := validateAge(-5)
var verr *ValidationError
if errors.As(err, &verr) {
    fmt.Printf("Поле: %s, значение: %v\n", verr.Field, verr.Value)
}
```
**Важно:** При возврате кастомной ошибки убедитесь, что возвращаете именно ошибку, а не `nil` конкретного типа (чтобы избежать ловушки с интерфейсом). Либо всегда возвращайте `nil` для успешного случая.
### 2.5.10. Паника и восстановление (panic/recover)

Хотя это не относится к обычным ошибкам, важно понимать, что в Go есть механизм **паники** — аварийной ситуации, которая прерывает нормальный поток выполнения. Паника обычно возникает при:

- выходе за границы слайса,
- разыменовании `nil`-указателя,
- явном вызове `panic()`.

Панику можно перехватить с помощью `recover()` внутри отложенной функции (`defer`). Это редко используется в обычном коде и больше подходит для:

- остановки программы при невосстановимых ошибках (например, неверная конфигурация),
- перехвата паники в горутинах, чтобы программа не упала полностью.

**Пример:**
```go
defer func() {
    if r := recover(); r != nil {
        fmt.Println("Восстановились после паники:", r)
    }
}()
panic("что-то пошло ужасно")
```
Однако **не стоит использовать панику для обработки обычных ошибок** — для этого есть возврат ошибок. Паника — это исключительная ситуация, после которой программа обычно не может корректно продолжать работу.

### 2.5.11. Игнорирование ошибок

Иногда можно встретить код, где ошибка игнорируется с помощью `_`:
```go
result, _ := doSomething()
```
Это допустимо только в том случае, если вы абсолютно уверены, что ошибки быть не может, или если это действительно неважно (например, при записи в буфер, который не может дать ошибку). В остальных случаях игнорирование ошибок — плохая практика. Если приходится игнорировать, лучше добавить комментарий с пояснением.
### 2.5.12. Ошибки в многопоточном коде

При работе с горутинами важно корректно передавать ошибки из горутины в основной поток. Часто используют каналы ошибок или `sync.WaitGroup` в сочетании с общей переменной ошибки (защищённой мьютексом). Но это уже продвинутая тема.
