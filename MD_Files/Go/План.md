## 1. Введение и настройка окружения

### 1.1. Установка Go

- Скачивание и установка с официального сайта.
    
- Проверка установки: `go version`.
    
- Настройка `PATH` (чтобы работала команда `go`).
    

### 1.2. Понимание GOPATH и модулей

- **GOPATH**: историческая структура, почему от неё ушли.
    
- **Go Modules**: современный способ управления зависимостями.
    
    - Инициализация модуля: `go mod init <module name>`.
        
    - Файлы `go.mod` и `go.sum`.
        
    - Версионирование зависимостей (semver).
        
    - Команды `go get`, `go mod tidy`, `go mod vendor`.
        

### 1.3. Первая программа

- Создание `main.go`.
    
- Структура: `package main`, `func main()`.
    
- Запуск: `go run main.go`.
    
- Компиляция: `go build`.
    

### 1.4. Редактор кода / IDE

- Рекомендации: VS Code с расширением Go, GoLand.
    
- Настройка автодополнения, форматирования, линтинга.
    

**Практика #1:** Установи Go, создай модуль `hello`, напиши программу, которая выводит "Hello, world!". Попробуй использовать внешнюю библиотеку (например, `rsc.io/quote`) и запусти код.

## 2. Основы синтаксиса и базовые конструкции

### 2.1. Переменные и константы

- Объявление: `var`, `:=`.
    
- Типы данных: `int`, `float64`, `bool`, `string`.
    
- Приведение типов.
    
- Константы: `const`, iota.
    

### 2.2. Управляющие конструкции

- Условный оператор `if` (с короткой записью).
    
- `switch` (без `break`, множественные условия).
    
- Циклы: только `for` (классический, как `while`, бесконечный).
    
- Операторы `continue`, `break`.
    

### 2.3. Функции

- Объявление: параметры, возвращаемые значения.
    
- Множественные возвращаемые значения.
    
- Именованные результаты.
    
- Функции как значения первого класса.
    
- Анонимные функции, замыкания.
    
- Отложенный вызов `defer` (механизм работы, порядок вызова).
    

### 2.4. Пакеты и импорты

- Структура пакета.
    
- Экспортируемые и неэкспортируемые идентификаторы (заглавная буква).
    
- Импорт с псевдонимом, точка, пустой импорт.
    

### 2.5. Обработка ошибок (начало)

- Ошибки как значения.
    
- Тип `error`.
    
- Создание ошибок: `errors.New`, `fmt.Errorf`.
    
- Простейшая проверка `if err != nil`.

## 3. Составные типы данных и работа с ними

### 3.1. Массивы

- Фиксированная длина.
    
- Объявление, инициализация.
    
- Передача в функции (копирование).
    

### 3.2. Слайсы (самое важное!)

- **Устройство слайса**: указатель на массив, длина (`len`), ёмкость (`cap`).
    
- Создание: `make`, литералы, срез массива.
    
- Оператор среза `s[low:high]`.
    
- Функции `append`, `copy`.
    
- Рост слайса: как и когда происходит перераспределение памяти.
    
- Связь с массивом под капотом.
    
- Слайсы как параметры функций (передаётся заголовок, но массив общий).
    

### 3.3. Мапы (map)

- **Устройство**: хеш-таблица, структура hmap, бакеты.
    
- Создание: `make`, литералы.
    
- Чтение и запись, проверка существования ключа.
    
- Удаление: `delete`.
    
- Итерация (порядок случаен).
    
- Важно: мапа — ссылочный тип, но передаётся как значение (указатель на hmap).
    
- Горутинобезопасность: чтение и запись из разных горутин (data race).
    

### 3.4. Структуры

- Определение типа `struct`.
    
- Поля с тегами (для сериализации, валидации).
    
- Сравнение структур.
    
- Анонимные структуры.
    
- Встраивание (embededding) — основа "наследования" в Go.
    

### 3.5. Указатели

- Операторы `&` и `*`.
    
- Указатели на структуры (автоматическая разыменовка).
    
- Когда использовать указатели (большие структуры, мутация).
    
- `new` vs `make`.

## 7. Углублённое изучение планировщика и сборщика мусора

### 7.1. Модель планировщика Go (GMP)

- G — goroutine, M — machine (поток ОС), P — processor (логический процессор).
    
- Работа планировщика: локальные и глобальная очереди горутин.
    
- Системные вызовы и блокировки: как P освобождается.
    
- Кооперативная и вытесняющая многозадачность в Go (до Go 1.14 и после).
    
- Как планировщик влияет на производительность.
    

### 7.2. Сборщик мусора (GC)

- Трёхцветная маркировка (tri-color mark-and-sweep).
    
- Фазы: Mark Setup, Marking, Mark Termination, Sweep.
    
- Конкурентный GC: как минимизирует stop-the-world.
    
- Настройка GC: переменная `GOGC` (формула целевой кучи).
    
- Инструменты для анализа GC: `GODEBUG=gctrace=1`, `pprof`.
    

### 7.3. Взаимодействие с планировщиком

- Как `go` запускает горутину.
    
- Блокирующие операции (каналы, мьютексы, syscall) и переключение контекста.
    
- `runtime.Gosched()` — добровольная передача управления.

### 8.1. Стек и куча

- Преобразования типов без копирования.

- Риски: нарушение типобезопасности, зависимость от архитектуры, несовместимость с GC.
    
- Когда оправдано: взаимодействие с C, оптимизация в критических местах.
    

### 8.3. Анализ памяти

- Профилирование памяти (heap профиль) с `pprof`.
    
- Поиск утечек памяти (горутины, неосвобождаемые ресурсы).

## 9. Принципы проектирования и архитектура

### 9.1. SOLID в Go

- **S**ingle Responsibility: пакеты и типы с одной ответственностью.
    
- **O**pen/Closed: через интерфейсы и композицию.
    
- **L**iskov Substitution: интерфейсы должны быть честными.
    
- **I**nterface Segregation: маленькие интерфейсы.
    
- **D**ependency Inversion: зависеть от абстракций.
    

### 9.2. Композиция вместо наследования

- Встраивание структур (embedding) как способ переиспользования.
    
- Проблемы с переопределением методов (Go не поддерживает классическое наследование).
    

### 9.3. Паттерны проектирования (с примерами на Go)

- **Порождающие**: Singleton (через `sync.Once`), Builder (с флюент-интерфейсом), Factory (функции, возвращающие интерфейсы).
    
- **Структурные**: Adapter (через интерфейсы), Decorator (функции высшего порядка), Facade (упрощённый интерфейс к сложной системе).
    
- **Поведенческие**: Strategy (интерфейс стратегии), Observer (каналы/коллбэки), Command (структура с методом Execute), Iterator (каналы).
    

### 9.4. Архитектурные стили

- **Чистая архитектура (Clean Architecture)**: слои (domain, usecase, infrastructure).
    
- **Луковая архитектура**, гексагональная (ports & adapters).
    
- Разделение на пакеты: избегаем циклических зависимостей.
    

### 9.5. Дизайн пакетов

- Правила именования пакетов (лаконично, без "utils").
    
- Организация internal пакетов.
    
- Экспортируемость и минимизация API.
    
- Тестируемость через интерфейсы.

## 19. Финальный проект

**Идея:** Разработать микросервис для сокращения ссылок (URL shortener) с функциями аналитики.

**Требования:**

- REST API (или gRPC) для создания коротких ссылок, редиректа, получения статистики.
    
- Авторизация по JWT (пользователи, регистрация).
    
- Хранение в PostgreSQL (ссылки, пользователи, переходы).
    
- Кеширование популярных ссылок в Redis.
    
- Асинхронное обновление статистики переходов через очередь (RabbitMQ/Kafka).
    
- Конфигурация через env.
    
- Структурированное логирование.
    
- Метрики Prometheus и трейсинг Jaeger.
    
- Graceful shutdown.
    
- Тесты: unit и интеграционные (с testcontainers).
    
- Документация API (swagger/openapi).
    
- Сборка в Docker, docker-compose для локального запуска.
    
- Профилирование и оптимизация под нагрузкой (написать скрипт нагрузки, снять профили, улучшить).
    

Этот проект объединит практически все изученные темы и даст реальный опыт разработки продакшн-сервиса.
