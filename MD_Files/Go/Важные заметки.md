## Примечание: Создание кастомных типов на основе встроенных

В Go вы можете создавать **новые типы** на основе существующих (встроенных или определённых в других пакетах). Это мощный механизм, который позволяет добавлять поведение (методы) к простым типам, сохраняя при этом **строгую типизацию**.
### 1. Синтаксис: `type NewType BaseType`
```go
type SimpleError string   // новый тип на основе string
type Counter int          // новый тип на основе int
type Flag bool            // новый тип на основе bool
```
После такого объявления `SimpleError` и `string` — это **разные, несовместимые типы**. Вы не можете просто присвоить строку переменной типа `SimpleError` без явного преобразования.
```go
var err SimpleError
err = "hello"              // ОШИБКА: cannot use "hello" (type string) as type SimpleError
err = SimpleError("hello") // ОК: явное преобразование
```
### 2. Зачем это нужно?

Главная причина — **добавление методов**. В Go нельзя добавлять методы к встроенным типам напрямую. Создав свой тип на основе встроенного, вы получаете возможность привязать к нему любые методы.
```go
type SimpleError string

// Теперь SimpleError реализует интерфейс error
func (e SimpleError) Error() string {
    return string(e)
}
```
### 3. Методы с получателем-значением vs получателем-указателем
|Получатель|Синтаксис|Особенности|
|---|---|---|
|Значение|`func (e SimpleError) Error() string`|Получает копию. Не может изменить оригинал. Подходит для маленьких неизменяемых типов.|
|Указатель|`func (e *SimpleError) Error() string`|Получает указатель на оригинал. Может изменять состояние. Используется, когда нужно мутировать или избежать копирования больших структур.|
### 4. Полный пример
```go 
package main

import "fmt"

// Создаём кастомный тип на основе string
type SimpleError string

// Реализуем интерфейс error
func (e SimpleError) Error() string {
    return string(e)
}

// Используем в функции, возвращающей error
func validateAge(age int) error {
    if age < 0 {
        return SimpleError("возраст не может быть отрицательным")
    }
    return nil
}

func main() {
    err := validateAge(-5)
    if err != nil {
        fmt.Println(err) // "возраст не может быть отрицательным"
    }
}
```
### 5. Когда использовать
- Когда нужно, чтобы ваш тип реализовывал какой-то интерфейс (например, `error`, `Stringer`, `json.Marshaler` и т.д.)
- Когда хотите добавить поведение к простому типу
- Когда нужна **типобезопасность** — чтобы разные смысловые категории не смешивались

```go
type UserID int
type OrderID int

func getUser(id UserID) { ... }
func getOrder(id OrderID) { ... }

func main() {
    var uid UserID = 42
    var oid OrderID = 42
    
    getUser(uid) // OK
    // getUser(oid) // ОШИБКА: cannot use oid (type OrderID) as type UserID
}
```
### 6. Важное отличие от псевдонимов (`type Alias = BaseType`)

Обратите внимание: `type SimpleError string` создаёт **новый тип**. А `type MyString = string` создаёт **псевдоним** (алиас), где `MyString` и `string` полностью взаимозаменяемы. Псевдонимы используются редко, в основном для совместимости при рефакторинге.

### 7. Где это применяется в стандартной библиотеке

Многие типы в Go построены именно так:

- `time.Duration` — это `int64`
- `os.FileMode` — это `uint32`
- `context.Context` — это интерфейс, но множество конкретных типов внутри — кастомные на основе структур

### Итог

|Концепция|Объяснение|
|---|---|
|`type Name BaseType`|Создаёт новый тип с тем же представлением в памяти, что и `BaseType`|
|Добавление методов|Новый тип может иметь методы, чего нельзя сделать с `BaseType`|
|Явное преобразование|Между новым и базовым типом нужно явное приведение `Name(value)`|
|Типобезопасность|Разные кастомные типы на одной основе не взаимозаменяемы|

Этот механизм — один из краеугольных камней Go, позволяющий строить типобезопасные абстракции без наследования.
